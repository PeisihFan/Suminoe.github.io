#回收技术
1.引用计数法

当一个对象被创建时，且将该对象实例分配给一个变量，该变量计数设置为1。

任何引用计数器为0的对象实例可以被当作垃圾收集。

缺点是无法检查出循环引用

例子

public class Main {
    public static void main(String[] args) {
        MyObject object1 = new MyObject();
        MyObject object2 = new MyObject();

        object1.object = object2;
        object2.object = object1;

        object1 = null;
        object2 = null;
    }
}

2.java 分代回收

所有新生成的对象首先都是放在年轻代的。年轻代大约按照8:1:1分成eden区，surivor0区，surviror1区，当minor GC(不一定需要eden满)时候，复制存活的对象到s0区，清空e区 ，当s0区满的时候，复制s0和e存活的对象到s1区，清空s0和eden区，再交换s0区和s1区，s1区保持为空，当当survivor1区不足以存放 eden和survivor0的存活对象时，就将存活对象直接存放到老年代。

若是老年代也满了就会触发一次Full GC，也就是新生代、老年代都进行回收
老年代内存比新生代也大很多(大概比例是1:2)，

持久代用于存放静态文件，如Java类、方法等

3.python 分代回收

python将所有的对象分成三代（0代、1代、2代）。新建对象都是0代对象。如果对象经历了一次垃圾回收后依然存活，就变成下一代对象。垃圾回收时，会扫描所有的0代对象，0代对象扫描次数到达一个阈值后，扫描1代对象，1代对象扫描次数到达另一个阈值后，扫描2代对象，如此循环。

4.标记清除法

标记-清除算法采用从根集合进行扫描，对存活的对象对象标记，标记完毕后，再扫描整个空间中未被标记的对象，进行回收。

#jvm工作原理

1.PC程序计数器(线程私有)

存储每个线程下一步将执行的JVM指令 每个线程都有一个自己的PC，以便完成不同线程上下文环境的切换。

2.JAVA虚拟机栈(线程私有)

每个方法被执行的时候都会同时创建一个栈帧（Stack Frame）用于存储局部变量表、操作数栈、动态链接、方法出口等信息。每一个方法被调用直至执行完成的过程就对应着一个栈帧在虚拟机栈中从入栈到出栈的过程。

3.本地方法栈(线程私有)

虚拟机使用到的本地方法服务。

4.Java堆(共享内存)

存储对象实例以及数组值的，
Java中所有通过new创建的对象的内存都在此分配。在这里实行回收机制

5.方法区(共享内存)

存储每一个类的结构信息，例如运行时常量池，成员变量和方法数据，构造函数和普通函数的字节码内容
