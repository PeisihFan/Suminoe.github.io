#单例模式

单例类只能有一个实例。

单例类必须自己创建自己的唯一实例。

单例类必须给所有其他对象提供这一实例。

懒汉模式

class Singleton {
    private static Singleton instance;

    private Singleton(){

    }

    public static synchronized Singleton getInstance(){//假如不加同步的话 多线程并行的时候就会创建多个实例
        if(instance==null){
            instance=new Singleton();

        }
        return instance;
    }
}

缺点：降低整个访问的速度，而且每次获取对象都要判断。

双重检查锁

只需要同步一次了，从而减少了多次在同步情况下进行判断所浪费的时间。它只是在第一次创建实例的时候同步，以后就不需要同步了。

public class Singleton {  
    //volatile保证了不同线程对这个变量进行操作时的可见性，即一个线程修改了某个变量的值，这新值对其他线程来说是立即可见的。  
    private volatile static Singleton singleton;

    private Singleton (){}  

    public static Singleton getSingleton() {  
        if (singleton == null) {  
            synchronized (Singleton.class) {  
                if (singleton == null) {  //防止有多个进程都进入了第一个if
                    singleton = new Singleton();  
                }  
            }  
        }  
            return singleton;  
    }  
}

饿汉模式

public class Singleton{
    //类加载时就初始化
    private static final Singleton instance = new Singleton();

    private Singleton(){}
    public static Singleton getInstance(){
        return instance;
    }
}

缺点:它不是一种懒加载模式（lazy initialization），单例会在加载类后一开始就被初始化
假如 Singleton 实例的创建是依赖参数或者配置文件的，在 getInstance() 之前必须调用某个方法设置参数给它，那样这种单例写法就无法使用了。

静态内部类‘

public class Singleton {  
    private static class SingletonHolder {  
        private static final Singleton INSTANCE = new Singleton();  
    }  
    private Singleton (){}  
    public static final Singleton getInstance() {  
        return SingletonHolder.INSTANCE; 
    }  
}

使用JVM本身机制保证了线程安全问题。

在该方法中，Singleton有一个静态内部类，内部类在外部类加载的时候并不会加载，只有在调用getInstance方法的时候加载SingletonHolder类。

#工厂模式

简单工厂

interface Shape{
    public void draw();
}

class A implements Shape{
    public void draw(){
        System.out.println("AAA");
    }
}
class B implements Shape{
    public void draw(){
        System.out.println("BBB");
    }
}
class C implements Shape{
    public void draw(){
        System.out.println("CCC");
    }
}
class SimpleFactory {
    public static Shape getShape(String type){
        Shape s=null;
        if(type.equals("A")){
            s=new A();
        }else if(type.equals("B")){
            s=new A();
        }else if(type.equals("C")){
            s=new A();
        }
        return s;

    }
}

工厂方法

简单来说就是每样产品都要为它添加一个工厂(最常用)

interface Shape{
    public void draw();
}

class A implements Shape{
    public void draw(){
        System.out.println("AAA");
    }
}
class B implements Shape{
    public void draw(){
        System.out.println("BBB");
    }
}
class C implements Shape{
    public void draw(){
        System.out.println("CCC");
    }
}

interface F{
    public Shape getShape();
}

class FA implements F{
    public Shape getShape(){
        return new A();
    }
}

class FB implements F{
    public Shape getShape(){
        return new B();
    }
}

class FC implements F{
    public Shape getShape(){
        return new C();
    }
}

抽象工厂

interface Char{
    public void getChar();
}

class A1 implements Char{
    public void getChar(){
        System.out.println("AAA");
    }
}

class B1 implements Char{
    public void getChar(){
        System.out.println("BBB");
    }
}


interface Num{
    public void getNum();
}

class A2 implements Num{
    public void getNum(){
        System.out.println("111");
    }
}

class B2 implements Num{
    public void getNum(){
        System.out.println("222");
    }
}


interface F{
    public Num printNum();
    public Char printChar(); 
}

class P1 implements F{
    public Num printNum(){
        return new A2();
    }

    public Char printChar(){
        return new A1();
    }
}
public class Factroy{
    public static void main(String args[]){
        F f;
        Num n;
        Char c;

        f=new P1();
        n=f.printNum();
        c=f.printChar();

        n.getNum();
        c.getChar();
    }
}
